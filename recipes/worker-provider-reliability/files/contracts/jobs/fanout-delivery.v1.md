# `fanout-delivery.v1`

The River `kind` is exactly `fanout-delivery.v1`. Its JSON payload is:

```json
{"parentId":"run-1","itemId":"item-1","idempotencyKey":"run-1:item-1"}
```

`parentId`, `itemId`, and `idempotencyKey` are non-empty strings. The
idempotency key is stored on the item and attempt record and is the provider
deduplication key; it is not generated by a retrying worker.

Payload-breaking changes use a new `.vN` kind. The Go worker calls the external
provider, persists its outcome, and only then notifies Elysia. Workers never
call Socket.IO or another realtime transport directly.
